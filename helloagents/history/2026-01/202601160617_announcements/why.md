# 变更提案: 公告功能（管理员发布 / 用户只读 / 未读自动弹出 / 已读标记）

## 需求背景

当前 Realms 提供了用户控制台与管理后台，但缺少一个“系统公告”能力：

- 管理员无法在系统内发布更新通知、维护窗口提示、重要变更说明
- 用户无法在登录后第一时间看到未读信息，容易错过关键变更

因此需要加入最小可用的公告模块：管理员发布公告，用户只读查看；未读公告在用户登录并进入控制台时自动弹出，并在用户确认后标记为已读。

## 目标与成功标准

- `root` 管理员可在管理后台创建公告，并可发布/撤回、删除
- 登录用户可查看公告列表与公告详情（只读）
- 用户登录进入控制台（`/dashboard`）时，如存在未读公告，自动弹窗展示
- 用户点击“我知道了”或打开公告详情后，会被标记为已读（同一公告对同一用户幂等）

## 变更范围

- 数据层：新增公告表与用户已读表
- 用户控制台：新增“公告”入口（列表/详情），以及 Dashboard 的未读公告弹窗
- 管理后台：新增“公告”入口（创建/发布/撤回/删除）
- 不影响 `/v1/*` 数据面 API
