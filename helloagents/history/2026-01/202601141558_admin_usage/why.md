# 变更提案：管理后台全局用量统计

## 背景

当前系统已基于 `usage_events` 记录并结算用量，并提供：
- Web 控制台 `/subscription`：展示当前登录用户的用量窗口汇总（5h/7d/30d）。
- JSON API `/api/usage/windows`、`/api/usage/events`：供用户用数据面 Token 查询自身用量（接口风格参考 new-api 的 usage 设计，但保持最小实现）。

但管理后台缺少**全局用量视图**，管理员无法快速了解全站消耗与正在进行的预留用量，影响运营与排障。

## 目标

1. **普通用户**：可查看自身用量（已有能力，保持兼容）。
2. **管理员（root）**：可在管理后台查看**全局用量**：
   - 5h/7d/30d 三个窗口的已结算/预留汇总；
   - （可选但实用）按用户统计的 Top 消耗排行，用于快速定位异常消耗来源。

## 非目标（本次不做）

- 不接入第三方计费系统或成本对账。
- 不新增/变更数据库 schema。
- 不提供跨租户/组织维度（本项目已单租户化）。

## 成功标准

- 管理后台新增入口 `/admin/usage`（仅 root 可访问），能稳定渲染全局用量汇总。
- 不影响现有 `/subscription` 与 `/api/usage/*` 行为与返回结构。
