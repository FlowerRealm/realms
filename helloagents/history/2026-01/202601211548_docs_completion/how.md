# 技术设计: 文档补全（根目录协作文档 + 知识库索引完善）

## 技术方案

### 核心技术
- Markdown 文档（UTF-8，无 BOM）

### 实现要点
- 根目录协作文档采用“可直接使用”的最小模板，避免空洞声明。
- 保持链接简单可靠：统一使用仓库内相对路径，确保 README ↔ 知识库 ↔ 根目录文档可互相跳转。
- `LICENSE` 在未确认类型前不写成开源许可，避免法律风险；待确认后再落地最终内容。

## 安全与性能

- **安全:**
  - 文档中的 Key/Token 一律使用占位符（如 `rlm_...`、`sk-***`）。
  - 变更完成后执行敏感信息扫描（简单 `rg` 规则）确保无真实密钥、Cookie、Authorization 等敏感信息。
- **性能:** 无影响（纯文档变更）

## 测试与部署

- **测试:**
  - 执行 `go test ./...`（确保文档变更不影响构建脚本/生成物）
  - 手动检查 Markdown 相对链接可用（README ↔ `helloagents/wiki/*` ↔ 根目录协作文档）
- **部署:** 无（文档变更）
