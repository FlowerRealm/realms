# 技术设计: 管理后台侧边栏溢出修复

## 技术方案

### 核心技术
- HTML 模板（Go `html/template`）
- CSS（Bootstrap 5 基础上做局部覆盖）

### 实现要点
- 目标是“最小变更修复”：不引入额外 JS，不改变路由/菜单结构。
- 侧边栏容器固定高度并隐藏溢出，避免侧边栏内容把页面整体撑高。
- 导航列表（`<ul>`）作为可滚动区域：
  - `flex: 1 1 auto` 占据侧边栏剩余空间
  - `min-height: 0` 允许在 flex 容器内正确收缩
  - `overflow-y: auto` 启用纵向滚动
  - 可选：`overscroll-behavior: contain` 减少滚动穿透；`-webkit-overflow-scrolling: touch` 优化移动端滚动

## 安全与性能
- **安全:** 仅样式/模板调整，不涉及鉴权/数据处理；不新增外部依赖与脚本。
- **性能:** 只增加少量 CSS 规则，影响可忽略。

## 测试与部署
- **测试:**
  1. 桌面端：缩小窗口高度到 600px/500px，确认侧边栏导航区域可滚动且底部标识可见。
  2. 移动端：宽度 < 768px 时展开侧边栏，确认内部可滚动且不影响主内容区。
  3. 回归检查：各管理页（如 `/admin/settings`、`/admin/users` 等）布局无错位。
- **部署:** 无额外步骤，随服务发布即可生效。
