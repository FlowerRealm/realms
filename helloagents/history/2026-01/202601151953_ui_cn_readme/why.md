# 变更提案: 用户可见界面与 README 全中文化

## 需求背景

当前项目的用户可见界面（Web 控制台 + 管理后台）与 `README.md` 中存在中英混用的固定文案，导致：

- 用户阅读负担增加（同一概念多种说法：如 Dashboard/Admin/Token 等）。
- 术语不统一，影响“Codex 中转站”场景下的可理解性与专业感。

本次变更目标是在不破坏任何接口/配置兼容性的前提下，将用户可见的固定文案统一为简体中文，并保留必要的专有名词与技术标识符。

## 变更内容

1. **Web 控制台与管理后台的固定文案中文化**
   - 导航、按钮、表格列名、状态标签、页脚等固定文本统一改为简体中文。
   - 术语按“中转站常用口径”统一（如：渠道/端点/凭证/授权账户/用量/订阅/模型等）。
2. **页面 Title 与提示文案中文化**
   - 浏览器标题栏、系统提示等固定字符串统一为中文（保留产品名 `Realms` 等）。
3. **`README.md` 中文化**
   - 将 README 中的非专有名词英文表述替换为中文，并统一术语与表达口径。

## 影响范围

- **模块:**
  - `internal/web`（用户控制台 SSR 模板与页面标题）
  - `internal/admin`（管理后台 SSR 模板与页面标题）
- **文件:**
  - `internal/web/templates/*.html`
  - `internal/admin/templates/*.html`
  - `internal/web/server.go`
  - `internal/admin/server.go`（仅标题等固定字符串）
  - `README.md`
- **不在范围:**
  - `.tmp/`（调研/临时代码）
  - `internal/upstream/`（上游指令与参考资料）
  - 代码标识符、API 路径与参数名、JSON 字段名、配置 key、环境变量名、命令行 flag（视为专有名词/技术标识符，保持不变）

## 核心场景

### 需求: 用户界面可读性
**模块:** `internal/web` / `internal/admin`
用户访问 Web 控制台与管理后台时，不再看到混杂的英文导航与表格列名，统一中文表达。

### 需求: 文档可读性
**模块:** `README.md`
用户按照 README 进行本地启动、配置与接入时，整体表达与术语一致，降低误解成本。

## 风险评估

- **风险:** 术语翻译不一致或不符合行业习惯，导致用户困惑。  
  **缓解:** 在 `how.md` 定义术语表与保留规则，并对目标文件做统一扫描校验。
- **风险:** 误改技术标识符（如环境变量名、配置 key、API 路径），造成用户复制粘贴失败。  
  **缓解:** 明确“专有名词/技术标识符不翻译”清单，并在修改后做差异复核与示例检查。
