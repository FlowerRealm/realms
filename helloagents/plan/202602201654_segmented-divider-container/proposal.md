# 变更提案：分段容器（外框 + 分隔线）

## 背景

当前前端为了增强“实线框”曾对 `card/table/alert` 等组件做了全局边框强化，但用户反馈希望：

- **不是每个元素外面都画线**
- **只在“块与块的分隔处”画线**
- 同时整个容器外侧仍需要一圈边框包住

并且分隔线要与现有“圆滑实线 + 角处留白”的风格一致，整体更轻。

## 目标

1. 提供一个可复用的 `div` 容器封装（React 组件 + CSS class）。
2. 容器具备：
   - 外框：4 条线段围框，角部留白（不闭合）
   - 内部分隔：仅在相邻块之间绘制分隔线（同一风格、两端留白）
3. 页面落地：优先在“系统设置”等典型多分区页面应用，验证效果后再扩展到更多页面。
4. Playwright 截图回归：覆盖页面并更新基线，确保后续迭代可反复修正。

## 方案概述

- 新增 `SegmentedFrame` 组件：将 children 分段包装为 `.rlm-segment`，容器使用 `.rlm-segmented`。
- CSS 实现：
  - `.rlm-segmented::before` 绘制外框 4 条线段
  - `.rlm-segment + .rlm-segment::before` 绘制内部水平分隔线
  - 所有参数由 `--rlm-frame-*` tokens 控制（线宽、颜色、角留白、圆角、分隔线 inset）
- 将全局“给所有组件都加线”的策略收敛回常规 `card` 细边框，避免过度装饰。

## 风险与回滚

- 风险：全局视觉变化较大（边框策略改变），需要依赖 Playwright 基线更新来锁定预期。
- 回滚：移除 `.rlm-segmented/.rlm-segment` 相关 CSS 与组件使用点；恢复原有全局边框策略（不建议）。
